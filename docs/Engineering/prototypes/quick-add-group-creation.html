Inline group creation in quick-add — when neither the product nor its group exists yet

One Group combobox unifies every case. As you type a group name the form previews exactly what will be minted. Empty = standalone (today). Pick existing = variant. New name = create the group + this product as its first variant, atomically.

⚠️
Why it must happen at first-create: a group is an abstract parent that can't hold stock, and a stock-holding product can't be converted into a group later ("Remove all stock lots before adding a variant"). So the group has to be born with its first variant, before the Take Stock count lands on it.

Back-end either way: a new atomic CreateGroupedProductCommand(groupName, variantName, unit, category?, location?) that mints the parent and first variant in one SaveChanges — mirroring how CreateVariantCommand fused create + link.
